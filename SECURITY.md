# Security Policy

## Supported Versions

Open Docs is a template; the `main` branch always carries the latest security
fixes. We recommend tracking the most recent release.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it privately using GitHub's
[private vulnerability reporting](https://github.com/shahnawaz-pabon/open-docs/security/advisories/new)
for this repository. If that is unavailable, open a minimal issue asking for a
private contact channel without disclosing details.

Please include:

- A description of the vulnerability and its impact
- Steps to reproduce, or a proof of concept
- Any suggested remediation

## Response Process

- We aim to acknowledge reports within **72 hours**.
- We will investigate and keep you informed of progress.
- Once a fix is ready, we will coordinate a release and credit you (unless you
  prefer to remain anonymous).

Thank you for helping keep Open Docs and its users safe.
